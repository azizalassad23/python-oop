NODE 04: ENCAPSULATION

Bayangkan kamu sedang membuat game multiplayer. Kamu sudah mendesain bos monster dengan darah (HP) 5000. Tiba-tiba, ada hacker (pemain iseng) yang menggunakan program ilegal untuk mengubah HP bos tersebut menjadi 1 dari luar sistem.

Sejauh ini, kode OOP yang kita buat sangat rentan terhadap serangan seperti itu. Mari kita lihat alasannya.

1. Masalah: Atribut Terbuka (Public)

Secara default, semua atribut di Python bersifat Public. Artinya, siapa saja bisa membaca dan mengubahnya dari luar Class dengan bebas.

📄 Contoh: Serangan Hacker
class Monster:
    def __init__(self):
        self.hp = 5000

bos = Monster()

# HACKER MENYERANG! 
bos.hp = 1  # Hacker bebas mengubah HP dari luar class!
print("HP Bos:", bos.hp)  # Output: 1 (Game rusak!)

2. Solusi: Menggembok Atribut (Private)

Untuk mencegah hacker menyentuh data kita dari luar, kita harus melakukan Encapsulation. Caranya: tambahkan dua garis bawah (underscore) di depan nama atribut.

📄 Contoh: Atribut yang Digembok
class Monster:
    def __init__(self):
        self.__hp = 5000  # Ditambah __ (Private)

bos = Monster()
print(bos.__hp) # ERROR! Akses diblokir.

3. GETTER: Jendela Melihat Data

Jika __hp digembok, bagaimana game menampilkan sisa darah ke layar? Kita buat Method perantara untuk mengambil data. Method ini disebut Getter.

💡 MENGAPA PAKAI RETURN, BUKAN PRINT?
Getter biasanya menggunakan return karena tugasnya melempar data rahasia ke luar agar bisa diolah (misal: diubah jadi grafik bar darah), bukan sekadar mencetaknya.
📄 Contoh: Fungsi Getter
class Monster:
    def __init__(self):
        self.__hp = 5000
    
    def cek_darah(self): # Ini Getter
        return self.__hp

bos = Monster()
print(bos.cek_darah())  # Output: 5000
